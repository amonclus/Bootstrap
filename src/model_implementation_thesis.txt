\subsection{Model implementation}\label{sec:model_implementation}

All models are implemented in Python using NetworkX to represent the underlying network.
The network is an undirected graph G = (V, E) with |V| = n nodes, where each node is
identified by an integer in {0, ..., n-1}. Simulation proceeds in synchronous discrete
rounds: at each round, all nodes update their state simultaneously based on the current
state of their neighbours.

A small initial set of nodes -- the seed set -- is marked as infected before the
simulation begins. Three seed-selection strategies are supported: random selection,
selection of the highest-degree nodes, and selection of the nodes with highest
k-core coreness. For statistical robustness, each experimental configuration is
repeated over multiple independent trials (typically 50-100), and the reported
metrics are averages across those trials.

The key output metrics collected per run are: the cascade fraction (the proportion
of nodes ever infected), the time to cascade (number of rounds until the system
stabilises), and a boolean flag indicating whether a large cascade occurred
(defined as cascade_fraction >= 0.5 for probabilistic models, or full network
infection for deterministic models). The critical seed size -- the minimum number
of seeds required to trigger a large cascade with probability at least 0.5 -- is
found via binary search over the number of seeds.


\subsubsection{State representation}

Every node in the simulation belongs to exactly one of the following compartments
at any given round:

  - Susceptible (S): the node has not yet been infected.
  - Infected (I): the node is currently infected and can spread to its neighbours.
  - Recovered (R): the node was previously infected and has recovered.

In the deterministic threshold models (Bootstrap Percolation and WTM), nodes
transition only from S to I and never recover. The simulation therefore maintains
a single growing set infected, and the complement susceptible shrinks monotonically
at each round.

In the probabilistic models with recovery (SIR, H3, and H6), three disjoint sets
are maintained: infected (current carriers), susceptible (never infected), and
recovered (removed from the dynamics). An additional set ever_infected tracks all
nodes that have ever entered the infected state and is used to compute the final
cascade fraction regardless of when nodes recovered. This is important because
in SIR-type dynamics the infected set may be empty before all nodes have been
processed, so cascade_fraction is defined as |ever_infected| / n.

The simulation state at the start of each round is thus fully characterised by
the partition (susceptible, infected, recovered) together with the round counter.


\subsubsection{Infection step}

The infection step is the central mechanism that differentiates the models and
is executed at the beginning of each round.

Bootstrap Percolation uses a hard absolute threshold k. A susceptible node v
becomes infected if and only if it has at least k infected neighbours:

    v becomes infected  <=>  |{u in N(v) : u infected}| >= k

The step is deterministic: given the current infected set, the set of newly
infected nodes is uniquely determined. All newly infected nodes are added to
the infected set simultaneously at the end of the round (synchronous update).

The Watts Threshold Model (WTM) uses a hard fractional threshold phi. Rather
than requiring an absolute count, a susceptible node v activates when the
fraction of its neighbours that are infected meets or exceeds phi:

    v activates  <=>  |{u in N(v) : u infected}| / deg(v) >= phi

Isolated nodes (deg(v) = 0) never activate. Like Bootstrap Percolation, WTM
is fully deterministic given the seed set.

The SIR model replaces the deterministic threshold with independent probabilistic
transmission. For each infected node u and each of its susceptible neighbours v,
v is independently infected with probability beta per round:

    P(v infected in round t) = 1 - (1 - beta)^|{u in N(v) : u infected}|

Equivalently, each infected-susceptible edge fires independently with probability
beta. This makes SIR memoryless at the level of individual contacts: a single
infected neighbour is sufficient to trigger infection with nonzero probability,
and having more infected neighbours increases the chance but in an independent,
non-reinforcing way.


\subsubsection{Recovery and termination}

In Bootstrap Percolation and WTM, infected nodes never recover. The simulation
terminates when a full round passes without any new node becoming infected,
i.e., when the cascade has reached a fixed point. The number of rounds elapsed
at that point is recorded as time_to_cascade.

In the SIR model and its hybrid variants (H3, H6), each infected node recovers
independently at the end of every round with probability gamma, transitioning to
the Recovered compartment. Recovered nodes are immune and take no further part
in the dynamics. The simulation terminates when the infected set becomes empty,
which is guaranteed to happen in finite time for any finite graph with gamma > 0.
The total number of rounds until termination is recorded as time_to_cascade.

In the SIS variant (not the focus of this work but included for completeness),
recovered nodes return to the Susceptible compartment rather than to an absorbing
Recovered state, allowing re-infection and the possibility of an endemic
equilibrium. To prevent infinite runs in the endemic regime, SIS simulations are
capped at a maximum of 500 rounds.

A run is classified as a large cascade if the cascade fraction exceeds 0.5
(i.e., more than half the network was ever infected). For Bootstrap Percolation
and WTM the threshold used instead is full network infection (cascade_fraction = 1),
reflecting the deterministic all-or-nothing nature of those models.


\subsubsection{Soft threshold models (H3 and H6)}

The Bootstrap Percolation and WTM models share a structural limitation: their
activation rule is a hard step function. A node that has k-1 (or phi*deg - 1)
infected neighbours has exactly zero probability of activating, while a node
that reaches the threshold activates with certainty. This discontinuity makes
the models analytically tractable but arguably unrealistic: in social and
biological contagion, having more infected contacts typically increases the
probability of infection continuously rather than switching abruptly.

To address this, H3 and H6 introduce soft (probabilistic) analogues of Bootstrap
Percolation and WTM respectively, while retaining SIR-type recovery dynamics.


H3 -- Probabilistic threshold (soft Bootstrap Percolation)

In H3, the hard threshold k of Bootstrap Percolation is replaced by a probability
that grows linearly with the number of infected neighbours m:

    P(v infected | m infected neighbours) = min(1, m * beta)

Each infected neighbour contributes an independent additive probability beta of
infecting v. This formula is a soft version of social reinforcement: the more
infected contacts a node has, the more likely it is to be infected, but the
transition is gradual rather than binary.

The model has a natural soft threshold at m* = 1/beta: when a susceptible node
has approximately 1/beta infected neighbours, its infection probability saturates
at 1 and it is effectively certain to be infected in that round, recovering the
all-or-nothing behaviour of Bootstrap Percolation. For m well below m*, the
probability is proportional to m*beta, and for m = 1 it reduces to P = beta,
which is identical to SIR. H3 thus interpolates smoothly between SIR (low beta
or low m) and Bootstrap Percolation (high beta or large m).

The infection step for H3 is implemented as follows: for each susceptible node v,
the number of infected neighbours m is counted; if m > 0, v is infected in the
current round with probability min(1.0, m * beta). Recovery follows standard
SIR dynamics: each infected node recovers independently with probability gamma
at the end of the round. The simulation terminates when the infected set is empty.

Parameters: beta (per-neighbour infection probability, in (0, 1]) and gamma
(recovery rate, in [0, 1]).


H6 -- Probabilistic fractional threshold (soft WTM)

H6 is to WTM what H3 is to Bootstrap Percolation: it replaces the hard fractional
threshold phi of WTM with a continuous probability that scales with the infected
fraction of a node's neighbourhood:

    P(v infected | m infected neighbours, deg(v) = d) = min(1, (m/d) / phi)

The infected fraction m/d is normalised by the threshold phi, so that the
probability reaches 1 exactly when the fraction of infected neighbours equals phi,
recovering the hard WTM rule at the saturation point. Below phi, the probability
is proportional to (m/d)/phi, providing soft reinforcement that is degree-aware:
nodes are penalised relative to how far their infected neighbourhood fraction falls
short of the threshold. Isolated nodes (d = 0) are excluded and never activate.

The relationship between H3 and H6 mirrors that between Bootstrap Percolation
and WTM: H3 uses absolute infected-neighbour counts while H6 normalises by
degree. As a consequence, H6 is more equitable across nodes of different
degrees -- a hub and a leaf node require the same infected fraction (not the same
absolute count) to be activated with the same probability. This degree-normalised
reinforcement is particularly relevant in heterogeneous networks where degree
distributions are broad.

The infection step for H6 proceeds identically in structure to H3: for each
susceptible node v with degree d > 0, m infected neighbours are counted; if m > 0,
v is infected with probability min(1.0, (m/d) / phi). Recovery and termination
are identical to H3 and standard SIR.

Parameters: phi (fractional threshold, in (0, 1]) and gamma (recovery rate,
in [0, 1]).
